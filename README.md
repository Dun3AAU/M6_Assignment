# M6 Assignment – Automated Weather Pipeline

An end-to-end automated data pipeline that collects weather forecasts, stores them in a SQLite database, generates bilingual poems using a large language model, and publishes the result to **GitHub Pages**.

Live site: [GitHub Pages](https://dun3aau.github.io/M6_Assignment/)

---

## Overview

Every day at 20:00 Danish time, two GitHub Actions workflows run automatically:

1. **Fetch Weather Data** – calls the [Open-Meteo API](https://open-meteo.com/) for three Danish locations and writes the hourly forecast data to a local SQLite database.
2. **Generate Poems and Update Pages** – reads tomorrow's weather data, asks the [Groq](https://groq.com/) LLM to produce a Danish and an English poem that compare the three locations and recommend the best spot for kitesurfing, then commits the output and deploys the site to GitHub Pages.

## Weather Variables

The following variables are fetched from the Open-Meteo API:

- **Temperature** at 2 m (°C)
- **Precipitation probability** (%)
- **Wind speed** at 10 m (m/s)
- **Wind direction** at 10 m (°)

## Setup

### Prerequisites

- Python 3.12+
- [uv](https://docs.astral.sh/uv/) package manager
- A [Groq API key](https://console.groq.com/)

### Install dependencies

```bash
uv sync
```

### Environment variables

| Variable | Description |
|----------|-------------|
| `GROQ_API_KEY` | Groq API key used by `generate_poems.py` |
| `GROQ_MODEL` | Groq model name (default: `llama-3.3-70b-versatile`) |
| `SQL_DB_PATH` | Path to the SQLite database (default: `data/weather.db`) |
| `WANDB_API_KEY` | Weights & Biases API key (required for online logging) |
| `WANDB_PROJECT` | W&B project name (default: `m6-assignment`) |
| `WANDB_ENTITY` | W&B entity/team name (optional) |
| `WANDB_MODE` | W&B mode (`online`, `offline`, `disabled`) |
| `WEAVE_DISABLED` | Set to `1` to disable Weave tracing |

Create a `.env` file or export the variables in your shell before running the scripts locally.

## Running Locally

### 1. Fetch weather data

```bash
uv run fetch.py
```

This fetches the next 2 days of hourly weather data for all three locations and stores it in `data/weather.db`.

### 2. Generate poems

```bash
uv run generate_poems.py
```

This reads tomorrow's weather from the database and writes:
- `outputs/poems.md` – Markdown with weather summary and poems
- `docs/index.html` – GitHub Pages HTML page

If W&B is configured, this command also logs artifacts for:
- weather input context (`rows`, `summary`, selected time window, prompts)
- optional SQLite snapshot (`weather.db`)
- raw Groq response + parsed JSON payload
- final generated outputs (`result.json`, `poems.md`, `index.html`)

You can target a specific time window by setting the `WEATHER_PERIOD` environment variable:

```bash
# Single hour
WEATHER_PERIOD="14:00" uv run generate_poems.py

# Time range
WEATHER_PERIOD="12:00-18:00" uv run generate_poems.py
```

## GitHub Actions Workflows

### `fetch-weather.yml`

Runs **daily at 19:00 UTC** (20:00 Copenhagen winter time).

Steps:
1. Checks out the repository
2. Installs Python 3.12 and `uv`
3. Runs `fetch.py` to update the database
4. Commits and pushes `data/weather.db`

### `generate-poems.yml`

Triggered automatically **after `fetch-weather.yml` completes**, or manually via `workflow_dispatch`.

Steps:
1. Checks out the repository
2. Installs Python 3.12 and `uv`
3. Runs `generate_poems.py` (requires `GROQ_API_KEY` secret)
4. Commits and pushes `docs/` and `outputs/`
5. Deploys `docs/` to GitHub Pages

> **Required secret:** Add `GROQ_API_KEY` to your repository's *Settings → Secrets and variables → Actions*.

## GitHub Pages

The site at `docs/index.html` is automatically updated by the CI pipeline and displays:

- The time window the poems cover
- A weather summary table (avg/min/max temperature, precipitation probability, wind speed)
- The Danish poem
- The English poem
- The kitesurfing recommendation
